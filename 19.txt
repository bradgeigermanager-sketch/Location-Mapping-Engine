Here’s a clean, modular multi‑point predictive system:

4. Core multi‑point predictive algorithms

State representation

- Stack all point states into one global state vector  
  \(\mathbf{S}(t) = [s1(t), s2(t), \dots, s_N(t)]^\top\)  
  where each \(s_i(t)\) includes 3D + 4+D brane components.

Independent model

- Predict each point with its own single‑point model.

Coupled linear model

\[
\mathbf{S}(t+\Delta t) = A \mathbf{S}(t)
\]

- \(A\) is a block matrix encoding interactions between points.

Interaction graph model

- For each point \(i\):

\[
si(t+\Delta t) = f\Big(si(t), \{sj(t)\}{j \in \mathcal{N}(i)}\Big)
\]

- \(\mathcal{N}(i)\) from interactionstructure.graphedges.

5. Multi‑point error and scoring

Per‑point error: use your existing trajectory prediction evaluation.

Interaction error example (for a pair \(i,j\)):

- Compare predicted vs actual brane comparison or distance:

\[
E{ij} = |D{\text{pred}}(i,j) - D_{\text{actual}}(i,j)|
\]

Aggregate error:

\[
E_{\text{multi}} =
\alpha \cdot \frac{1}{N}\sumi Ei +
(1-\alpha) \cdot \frac{1}{M}\sum{(i,j)} E{ij}
\]

- \(E_i\): per‑point total error  
- \(E_{ij}\): interaction error  
- \(\alpha\): weight between individual and relational accuracy  

Weights inside \(E_i\) come from the metric profile.
