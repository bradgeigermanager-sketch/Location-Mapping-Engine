Below is the complete, atomic, schema‑driven system for trajectory similarity in the LME framework.  
This includes:

- A Trajectory Similarity Record Schema  
- A Trajectory Similarity Calculation Schema  
- The mathematical algorithms used to compute similarity across:  
  - 3D paths  
  - 4D separation evolution  
  - 5D+ brane‑space evolution  
- A unified similarity score driven by your metric profile  

Everything is modular, DAG‑safe, and consistent with the existing LME architecture.

⭐ 3. Trajectory Similarity Algorithms (Complete Set)

A. Temporal alignment

1. Dynamic Time Warping (DTW)
Aligns trajectories with different sampling rates or speeds.

Cost matrix:

\[
C(i,j) = \text{pair\cost}(Pi, Q_j)
\]

DTW recurrence:

\[
D(i,j) = C(i,j) + \min(D(i-1,j), D(i,j-1), D(i-1,j-1))
\]

2. Segmentwise alignment
Compare segment‑to‑segment using average metrics.

3. Synchronized alignment
Compare only matching timestamps.

---

B. Per‑pair cost function

For aligned samples \(Pi\) and \(Qj\):

\[
\text{pair\_cost} = 
\gamma3 D{3D}^2 +
\gamma4 D4^2 +
\gammas \sum{k=5}^{n} (\Delta d_k)^2 +
\gammaa \sum{k=5}^{n} (\Delta \theta_k)^2 +
\gammad \sum{k=5}^{n} (\dot{d}k^2 + \beta \dot{\theta}k^2)
\]

All weights come from the metric profile.

---

C. Similarity components

1. 3D similarity
Inverse of average 3D distance:

\[
S{3D} = \frac{1}{1 + \overline{D{3D}}}
\]

2. 4D similarity
\[
S{4D} = \frac{1}{1 + \overline{D4}}
\]

3. 5D+ static similarity
\[
S{5+}^{\text{static}} = \frac{1}{1 + \sqrt{\sumk (\Delta d_k)^2}}
\]

4. 5D+ dynamic similarity
\[
S{5+}^{\text{dynamic}} = \frac{1}{1 + \sqrt{\sumk (\dot{d}k^2 + \beta \dot{\theta}k^2)}}
\]

---

D. Total similarity score

\[
S_{\text{total}} =
\gamma3 S{3D} +
\gamma4 S{4D} +
\gammas S{5+}^{\text{static}} +
\gammad S{5+}^{\text{dynamic}} -
\lambda \cdot \text{temporal\alignment\cost}
\]

Where:

- \(\gamma\) weights come from the metric profile  
- \(\lambda\) penalizes temporal misalignment (DTW warping)
